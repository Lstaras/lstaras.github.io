三者的意义

总的来说三者的关系：

URI

URL

URN

URL和URN都是URI的子集，且二者也有交集。

URI 统一资源标识符

  • 组成

    1. 资源命名机制
    2. 存放资源的主机名
    3. 资源路径

  • 例子
    http:// www.abc.com/abc/indec.html #abc

  • 抽象
    用来确定一个人的某种或几种信息

URL 统一资源定位符

URL目前在B-S架构中最为流行。

  • 组成

    1. 协议
    2. Ip地址（端口）
    3. 具体地址（目录/文件）

  • 例子
    http:// www.abc.com/abc/index.html
    在Web服务器可解析的URL中可能会有省略的情况，如：
    http:// www.abc.com
    http:// www.abc.com/
    http:// www.abc.com/abc
    http:// www.abc.com/abc/
    以上均省略了服务器事先设置的默认文件，如：/index.html /default.htm…

  • 抽象
    一个人的住址

URN 统一资源命名符

  • 组成

    1. 命名空间
    2. 资源名

  • 例子
    www.abc.com/abc/index.html #abc

  • 抽象
    一个人的身份证号
